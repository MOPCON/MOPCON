自從接觸到 IoT 後，身為一個硬體人的我開始覺得，沒有軟體思維真的是萬萬不行。我和 IoT 的緣分，最早是從學習 ZigBee 開始，而後我又發現光是會寫晶片韌體，真的是萬萬不行。後來，我遇到了 Node.js，試著用它來寫 ZigBee 驅動程式，在完成的那一天我又驚覺萬萬不行了。隨後，開啟了我學習 Web 前後端的道路。一頭栽進軟體領域，直到今天我還是像個嬰兒般，走得跌跌撞撞。我希望在這場演講介紹一點熱門的 IoT 通訊協定，再穿插一點我的學習感想，邀請大家在這 50 分鐘裡，和我一起夢遊仙境。

李健榮(Simen)

Tera - 無限容量、無限想像

對使用者來說，儲存容量是一個功能參數，而不是功能本身。使用者要的是資料存得好、拿得到，而不是在乎本地或雲端的儲存容量有多大、網路有多快。Tera 的理念是資料的 hyperconnectivity，也就是說所有資料儲存透過網路融為一體，使用者不用去擔心資料存在本地或雲端、網路是否有連線或有多快。Tera 首先藉由本地和雲端儲存的無縫連結，排除了第一個問題：所有的程式自動看到本地和雲端一體的儲存空間，而不用再去區分；再來藉由人工智慧調校過的本地快取，預測使用者將會需要哪些資料，而把它們保存在本地，以減少網路存取量以及延遲。 Tera 是和沛團隊過去五年重要的研發成果。它讓 Linux 電腦可以掛載 HCFS (Hopebay Cloud Filesystem)，使本地儲存空間看來跟雲端一樣大。本地儲存變成快取，讓存取效能不會受到網路延遲的影響。後端可以使用 Google Drive, SWIFT, AWS S3 等雲端儲存服務。目前 Tera 可以用在 Android 4.4 - 8.0 的手機上。

翟本喬
